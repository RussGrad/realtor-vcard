Как сделать визитку доступной по ссылке «везде»
=========================================

1) Соберите данные для интернета
   - Откройте кабинет, войдите по паролю, заполните визитку и сохраните.
   - Нажмите кнопку «Скачать для сайта (published.json)» — скачается файл
     realtor-vcard-published.json

2) Загрузите сайт на хостинг
   Нужно выложить ВСЮ папку проекта (или сборку): index.html, cabinet.html,
   папки css/, js/ и т.д.

3) Обязательно добавьте файл с данными
   Загрузите realtor-vcard-published.json В ОДНУ ПАПКУ с index.html
   (имя файла должно быть ровно: realtor-vcard-published.json).

   После этого любой по ссылке увидит вашу визитку с вашими текстами и фото
   (без доступа к кабинету — только просмотр).

4) Чем можно пользоваться для бесплатного хостинга статики
   - GitHub Pages
   - Netlify (перетащить папку)
   - Cloudflare Pages
   - Vercel (static)

5) Про ссылку для клиентов
   Делитесь адресом вида: https://ваш-домен.com/index.html
   или корнем сайта, если хостинг открывает index.html автоматически.

6) Про кабинет на сервере
   Адрес cabinet.html защищён паролем из js/realtor-cabinet-auth-config.js
   Перед публикацией смените пароль в этом файле.

7) Важно
   Правки только в браузере (localStorage на вашем ПК) для других людей не
   видны. Чтобы обновить сайт для всех — снова скачайте published.json,
   замените файл на хостинге и при необходимости очистите кэш (Ctrl+Shift+R).


────────────────────────────────────────
Вариант A — GitHub Pages
────────────────────────────────────────

Подойдёт, если папка realtor-vcard — корень репозитория (внутри только сайт).

1. Создайте репозиторий на github.com, загрузите туда содержимое папки
   realtor-vcard (index.html, cabinet.html, css/, js/, и т.д.).

2. Добавьте в репозиторий файл realtor-vcard-published.json
   (скачайте из кабинета кнопкой «Скачать для сайта», переименовать не нужно).

3. Репозиторий → Settings → Pages:
   - Build and deployment: Source = «Deploy from a branch»
   - Branch = main (или master), папка = / (root)
   - Save

4. Через 1–3 минуты сайт будет по адресу:
   https://ВАШ-ЛОГИН.github.io/ИМЯ-РЕПО/
   Например: https://ivanov.github.io/realtor-vcard/

5. Если кабинет или json не открываются — проверьте, что все файлы в корне
   репозитория, пути относительные (./) как у вас сейчас.

6. Обновление визитки: закоммитьте новый realtor-vcard-published.json
   и при необходимости изменения в js/ — push в main.


────────────────────────────────────────
Вариант B — Vercel
────────────────────────────────────────

1. Зарегистрируйтесь на vercel.com (удобно войти через GitHub).

2. New Project → Import ваш репозиторий с папкой визитки.

3. Настройки проекта:
   - Framework Preset: Other (или «No Framework»)
   - Root Directory: если репозиторий один и сайт в корне — оставьте пустым;
     если монорепозиторий — укажите подпапку (например realtor-vcard).
   - Build Command: оставьте пустым
   - Output Directory: оставьте пустым (корень = статика)

4. Environment variables не нужны для этой визитки — Deploy.

5. После деплоя Vercel даст ссылку вида:
   https://имя-проекта.vercel.app

6. Файл realtor-vcard-published.json должен лежать в том же корне проекта,
   который деплоится (добавьте в git и сделайте push, либо загрузите через
   репозиторий вместе с остальными файлами).

7. Свой домен: Project → Settings → Domains (по желанию).

8. Обновление: новый commit с обновлённым published.json → Vercel пересоберёт
   сам (обычно за секунды).


────────────────────────────────────────
GitHub vs Vercel — коротко
────────────────────────────────────────

GitHub Pages — проще, если уже пользуетесь GitHub; бесплатно, стабильно.
Vercel — быстрый деплой из Git, preview на каждый PR, удобные домены.

Для статической визитки оба варианта подходят одинаково; выберите по вкусу.
